**Creer `scripts/audit-code.ts`** — un script d'analyse statique qui scanne tout `src/` pour les patterns non couverts par ESLint et knip. Le script utilise `tinyglobby` (deja dans devDependencies) et `node:fs`.

**Structure du script :**

```typescript
// scripts/audit-code.ts
// Usage: npx tsx scripts/audit-code.ts
// Produit un JSON stdout avec les findings pour consolidation en AUDIT.md
```

**Interface Finding (adapter le pattern de audit-full.ts) :**

```typescript
interface Finding {
  severity: 'Critical' | 'Warning' | 'Info'
  category: 'Securite' | 'Performance' | 'DeadCode' | 'TypeScript' | 'CSS' | 'Config'
  file: string
  line: number
  description: string
  suggestion: string
}
```

**Checks a implementer (6 categories) :**

1. **SECURITE — Routes admin sans requireAdmin() :**
   - Scanner tous les fichiers `src/app/api/admin/**/route.ts`
   - Verifier que chaque fichier contient l'appel `requireAdmin()` ou `requireAdmin(`
   - Severite: Critical si absent

2. **SECURITE — Routes API sans validation Zod :**
   - Scanner `src/app/api/**/route.ts` (toutes les routes, admin et public)
   - Pour chaque route avec un handler POST/PUT/PATCH, verifier la presence de `.parse(` ou `.safeParse(` ou `schema` import
   - Severite: Warning si POST/PUT sans validation

3. **SECURITE — Security headers dans next.config.ts :**
   - Lire `next.config.ts` et verifier la presence de `headers` (la fonction async headers())
   - Verifier les headers : `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Content-Security-Policy`
   - Severite: Warning si `headers` absent, Info pour chaque header individuel manquant

4. **SECURITE — Inline styles avec potentiel XSS :**
   - Scanner tous les `.tsx` pour `style={{` suivi d'une variable (pas une valeur literale)
   - Les `style={{ objectFit: 'cover' }}` avec valeurs literales sont OK (Info seulement)
   - Les `style={{ width: variable }}` avec une variable dynamique sont Warning
   - Severite: Info pour literals, Warning pour dynamiques

5. **PERFORMANCE — await dans boucle (potentiel N+1) :**
   - Scanner tous les `.ts` et `.tsx` pour le pattern : une ligne contenant `for (`, `for(`, `.forEach(`, ou `.map(` suivie dans les 5 lignes suivantes d'un `await` contenant `supabase` ou `fetch`
   - Severite: Warning

6. **PERFORMANCE — console.log en production :**
   - Scanner tous les fichiers `src/**/*.{ts,tsx}` pour `console.log(`
   - Exclure les fichiers dans `scripts/` et `__tests__/`
   - `console.error` et `console.warn` sont OK (pattern etabli, ne pas reporter)
   - Severite: Info

7. **TYPESCRIPT — catch sans typage :**
   - Scanner pour `catch (` suivi d'un bloc qui n'a PAS `instanceof Error` ni `as Error` dans les 3 lignes suivantes
   - Pattern a detecter : `catch (err)` ou `catch (error)` ou `catch (e)` puis le corps n'utilise pas `instanceof Error`
   - Severite: Info

8. **TYPESCRIPT — Fonctions async sans try/catch :**
   - Scanner les handlers de route (`export async function GET`, `POST`, `PUT`, `PATCH`, `DELETE`) dans `src/app/api/**`
   - Verifier que le corps de la fonction contient `try {`
   - Severite: Warning si absent (erreur non catchee = 500 generique)

9. **CSS — Classes inline style={{}} dans les composants :**
   - Compter les occurrences de `style={{` dans tous les `.tsx`
   - Reporter chaque occurrence avec file:line
   - Severite: Info (convention CSS Modules, mais pas un bug)

10. **CONFIG — next.config.ts remotePatterns trop large :**
    - Lire `next.config.ts`, verifier si `remotePatterns` utilise `hostname: '**'` (trop permissif)
    - Verifier que le pattern est restreint a `**.supabase.co` ou plus specifique
    - Severite: Warning si trop permissif

**Output du script :**

Le script imprime les findings en JSON sur stdout :

```typescript
// A la fin du script
console.log(JSON.stringify(results, null, 2))
// Aussi imprimer un resume sur stderr
const criticals = results.filter(r => r.severity === 'Critical').length
const warnings = results.filter(r => r.severity === 'Warning').length
const infos = results.filter(r => r.severity === 'Info').length
console.error(`\nAudit termine: ${criticals} Critical, ${warnings} Warning, ${infos} Info`)
process.exit(criticals > 0 ? 1 : 0)
```
