**1. [Rule 3 - Blocking] @supabase/ssr@0.9.0 installé sans fichiers dist/**
- **Detecte durant :** Task 1 — premiere execution des tests
- **Probleme :** `@supabase/ssr@0.9.0` (tres recente, ~avril 2026) installee avec seulement des fichiers `.d.ts` dans `dist/` — aucun fichier `.js` compile. Erreur : `Failed to resolve entry for package "@supabase/ssr"`

- **Fichiers modifies :** package.json, package-lock.json
- **Commit :** 13dc18a (inclus dans le commit Task 1)

**2. [Rule 3 - Blocking] @supabase/supabase-js@2.100.0 installé sans dist/index.cjs**
- **Detecte durant :** Task 1 — apres fix du package ssr
- **Probleme :** `@supabase/supabase-js@2.100.0` installee avec seulement `dist/umd/` — pas de `dist/index.cjs` ni `dist/index.mjs`. Erreur : `Cannot find module '.../dist/index.cjs'`
- **Correction :** Downgrade vers `@supabase/supabase-js@2.50.0` (version stable)
- **Fichiers modifies :** package.json, package-lock.json
- **Commit :** 13dc18a (inclus dans le commit Task 1)

**3. [Rule 1 - Bug] Cookie format incorrect dans auth helper**
- **Detecte durant :** Task 2 — tests avec JWT retournaient 401 au lieu de 200
- **Probleme :** `auth.ts` encodait la session en `Buffer.from(json).toString('base64')` sans le prefixe `"base64-"`. `@supabase/ssr@0.6.1` cherche la valeur brute JSON ou la valeur prefixee `"base64-"` pour decoder. Sans prefixe, la valeur base64 est traitee comme JSON brut, ce qui echoue.
- **Correction :** Remplace l'encodage base64 par `encodeURIComponent(json)` — passe le JSON directement URL-encode, reconnu comme valeur brute valide par `@supabase/ssr`
- **Fichiers modifies :** src/__tests__/integration/helpers/auth.ts
- **Commit :** c260a22
