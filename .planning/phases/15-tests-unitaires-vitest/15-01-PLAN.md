**Code a ajouter :**

```typescript
// -------------------------------------------------------------------------
// timeout (D-07)
// -------------------------------------------------------------------------

describe('timeout (D-07)', () => {
  beforeEach(() => {
    vi.useFakeTimers()
  })

  afterEach(() => {
    vi.useRealTimers()
  })

  it('produit une erreur apres 3 retries sur AbortError (timeout 30s)', async () => {
    // Mock fetch pour resolveImagePart (URL path)
    vi.stubGlobal('fetch', vi.fn().mockResolvedValue({
      ok: true,
      arrayBuffer: () => Promise.resolve(new ArrayBuffer(8)),
      headers: new Headers({ 'content-type': 'image/jpeg' }),
    }))

    // AbortError est retryable dans isRetryableError() — 3 echecs attendus
    const abortErr = new Error('The operation was aborted')
    abortErr.name = 'AbortError'
    mockGenerateContent.mockRejectedValue(abortErr)

    const service = new NanoBananaService()
    const generatePromise = service.generate({
      modelName: 'Milano',
      fabricName: 'Velours Bleu',
      viewType: 'front',
      sourceImageUrl: 'https://example.com/image.jpg',
    })

    // Avancer tous les timers pour bypasser sleep() dans le retry loop
    await vi.runAllTimersAsync()

    await expect(generatePromise).rejects.toThrow()
    expect(mockGenerateContent).toHaveBeenCalledTimes(3)
  })
})
```

**Fallback si `vi.runAllTimersAsync()` ne fonctionne pas avec le pattern async :** Retirer `vi.useFakeTimers()`/`vi.useRealTimers()` et laisser le test prendre ~3.4s reels (identique au test `throw apres 3 echecs consecutifs retryables` existant qui prend 3108ms). Dans ce cas le test fonctionne mais est lent.
