## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Fix encodage cookie dans helpers/auth.ts**
- **Found during:** Task 1 — tous les tests authentifiés retournaient 401
- **Issue:** L'helper encodait la valeur de cookie en `Buffer.from(json).toString('base64')`. Or `@supabase/ssr` lit les cookies via `getItem` qui n'applique de décodage base64 que si la valeur commence par le préfixe `base64-` (encodage base64url). Un cookie encodé en base64 standard sans préfixe était interprété comme une chaîne JSON invalide → session introuvable → 401.
- **Fix:** Remplacé par `encodeURIComponent(cookieValue)` — la valeur JSON est passée directement via URL encoding, compatible avec le parsing HTTP des cookies et la lecture directe de `@supabase/ssr`.
- **Files modified:** `src/__tests__/integration/helpers/auth.ts`
- **Commit:** 6c21bc7

**2. [Rule 3 - Blocking] Création .env.test.local dans le worktree**
- **Found during:** Task 1 — les variables `TEST_ADMIN_EMAIL` et `TEST_ADMIN_PASSWORD` manquaient
- **Issue:** Le worktree ne dispose pas des fichiers gitignorés du répertoire principal. Le setup `dotenv({ path: '.env.test.local' })` cherche le fichier dans le CWD du worktree.
- **Fix:** Créé `.env.test.local` dans le worktree avec les 5 variables nécessaires (URL, anon key, service role key, credentials admin).
- **Files modified:** `.env.test.local` (worktree uniquement, gitignored)

**3. [Rule 1 - Bug] Correction assertions middleware tests**
- **Found during:** Task 2 — 2 tests middleware échouaient avec "null and string is invalid"
- **Issue:** `expect(null).not.toContain('/admin/login')` est invalide — `toContain` ne fonctionne pas sur `null`.
- **Fix:** Remplacé par `expect(location).toBeNull()` pour les routes sans redirect et `if (location !== null) { expect(location).not.toContain(...) }` pour le test /admin/login.
- **Files modified:** `src/__tests__/integration/routes/simulate.integration.test.ts`

## Architecture Notes

- Routes `bulk-validate` et `bulk-publish` utilisent **PUT** (pas POST) et acceptent `{ visual_ids: string[] }` (pas `{ ids }`) — différent de ce que le plan décrivait en commentaire.
- La contrainte UNIQUE `(model_image_id, fabric_id)` dans `generated_visuals` impose un mapping soigneux des couples de test pour éviter les conflits avec les données seed.
- Le mock Sharp provider (`src/lib/ai/mock.ts`) génère un buffer JPEG réel (9150 octets) en ~5ms — les tests simulate ne sont pas bridés par la performance IA.
- Le rate-limiter de `/api/simulate` est une `Map<ip, {count, resetAt}>` en mémoire. Les tests utilisent des IPs distinctes (`x-forwarded-for`) pour isoler les compteurs entre suites.
- Le test rate-limit (6 appels consécutifs) s'exécute en < 500ms avec le mock Sharp.
