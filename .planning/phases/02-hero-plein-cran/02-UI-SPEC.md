## Spacing Scale

Tokens CSS définis dans `src/app/globals.css` — utiliser exclusivement ces tokens.

| Token CSS | Valeur | Usage dans le Hero |
|-----------|--------|--------------------|
| `--spacing-xs` | 4px | Espaces micro (icône/badge internes) |
| `--spacing-sm` | 8px | Gap entre badge et H1 |
| `--spacing-md` | 16px | Gap entre H1 et sous-titre |
| `--spacing-lg` | 24px | Gap entre sous-titre et CTA |
| `--spacing-xl` | 32px | Padding latéral mobile (`--container-padding-mobile` = 24px, voir exception) |
| `--spacing-2xl` | 48px | Padding latéral desktop (`--container-padding-desktop`) |

Exceptions :
- `--spacing-section` = 112px — token layout dédié (64px header-height + 48px breathing room) — hors échelle standard, usage réservé à la compensation header hero
- Padding latéral mobile : `var(--container-padding-mobile)` = 24px (token dédié, non dans l'échelle standard)
- Padding latéral desktop : `var(--container-padding-desktop)` = 48px
- Padding latéral large : `var(--container-padding-large)` = 64px (>= 1280px)
- Hauteur header réservée : `var(--header-height)` = 64px (le contenu hero ne doit pas débuter sous 64px)
- Indicateur de scroll : positionné à `var(--spacing-2xl)` (48px) du bas du hero

## Typography

Tous les tokens sont définis dans `src/app/globals.css`. Utiliser uniquement les valeurs déclarées.

2 poids autorisés : **400** (corps, détails) et **700** (titres, labels uppercase).

| Rôle | Token CSS | Taille | Poids | Line-Height | Usage dans le Hero |
|------|-----------|--------|-------|-------------|--------------------|
| H1 desktop | `--font-size-display` | 56px (3.5rem) | 700 | 1.2 | Titre principal au-dessus de >=1024px |
| H1 mobile | `--font-size-hero-mobile` | 36px (2.25rem) | 700 | 1.2 | Titre principal en dessous de 1024px |
| Sous-titre | `--font-size-lg` | 18px (1.125rem) | 400 | 1.6 | Paragraphe bénéfice sous le H1 |
| Badge + CTA + Scroll | `--font-size-sm` | 14px (0.875rem) | 700 (badge, CTA) / 400 (scroll) | 1 | Badge "Visualisation par IA", libellé CTA, texte "Défiler" |

Règles :
- Letter-spacing H1 display : `-0.02em` (token charte)
- Letter-spacing badge et CTA : `0.05em` (labels uppercase)
- Jamais `#000000` — toujours `var(--color-text)` pour le texte sombre ; blanc `#FFFFFF` sur fond sombre hero

## Color

Palette issue de `src/app/globals.css`. 60/30/10 adapté au contexte hero (fond sombre).

| Rôle | Token CSS | Valeur | Usage |
|------|-----------|--------|-------|
| Dominant (60%) — fond hero | fond couleur unie warm + overlay | `#2C2418` (warm dark) + `rgba(0,0,0,0.55)` overlay | Surface de fond du hero, crée le contexte sombre |
| Secondary (30%) — texte | `#FFFFFF` | blanc | H1, sous-titre, indicateur scroll — tout le texte sur fond sombre |
| Accent (10%) | `--color-primary` / `--color-primary-dark` | `#E49400` / `#845400` | Badge IA (pill dorée), gradient CTA uniquement |
| Destructive | `--color-error` | `#BA1A1A` | Aucune action destructive dans cette phase |

Accent réservé pour :
1. Fond du badge pill "Visualisation par IA" — `--color-primary` à 90% opacité
2. Gradient du bouton CTA — `linear-gradient(135deg, var(--color-primary-dark), var(--color-primary))`

Aucun autre élément du hero n'utilise la couleur accent.

### Overlay du fond hero

- Fond de base : couleur unie warm `#2C2418` (placeholder jusqu'à l'image réelle)
- Overlay sombre : `rgba(0, 0, 0, 0.55)` — token `--color-overlay` de globals.css
- Cette intensité garantit un ratio de contraste WCAG AA (blanc sur fond foncé ≥ 4.5:1)
- Quand l'image réelle sera intégrée : même overlay `rgba(0,0,0,0.55)` appliqué par-dessus via `background-image` CSS (technique retenue — D-03)

### Technique image future (D-03)

`background-image` CSS avec `background-size: cover; background-position: center` — pas de `next/image` fill, pour éviter la complexité SSR sur un fond décoratif sans valeur sémantique.

## Animation d'entrée (Framer Motion)

| Propriété | Valeur | Source |
|-----------|--------|--------|
| Librairie | Framer Motion | CONTEXT.md D-13 |
| Éléments animés | Bloc contenu hero complet (badge + H1 + sous-titre + CTA ensemble) | CONTEXT.md D-11 — fade simultané |
| Animation | `opacity: 0 → 1`, `y: 20 → 0` | Fade-in avec légère montée |
| Durée | `400ms` | CONTEXT.md D-12 — cohérent avec `--transition-smooth` |
| Easing | `ease-in-out` | CHARTE-GRAPHIQUE.md règle générale |
| Délai | `0ms` — déclenché immédiatement au montage | — |
| `prefers-reduced-motion` | `duration: 0` si `prefers-reduced-motion: reduce` | CONTEXT.md discretion — accessibilité |

Implémentation : wrapper `<motion.div>` avec `initial={{ opacity: 0, y: 20 }}` et `animate={{ opacity: 1, y: 0 }}` + `transition={{ duration: 0.4, ease: 'easeInOut' }}`. Respecter `useReducedMotion()` de Framer Motion pour désactiver si besoin.

## States & Interactions

| État | Déclencheur | Comportement |
|------|-------------|--------------|
| Initial (page load) | Montage composant | Fade-in 400ms du bloc contenu (opacity 0→1, y 20→0) |
| Scroll actif | `scrollY > 0` | ScrollIndicator : `opacity: 0` via transition 300ms |
| Scroll nul | `scrollY === 0` | ScrollIndicator : `opacity: 1` |
| Hover CTA | Survol bouton | `translateY(-2px)` + `box-shadow: var(--shadow-md)`, 300ms ease |
| Focus CTA (clavier) | Tab focus | Outline visible `2px solid var(--color-primary)` + `outline-offset: 4px` |

SSR safety : `useState(false)` par défaut pour `isScrolled`, jamais d'accès à `window` dans l'initialisation (pattern établi Phase 01 — `src/components/public/Header/Header.tsx`).
