## Composants visuels — Contrat détaillé

### Rangée de thumbnails `.thumbnailRow`

- **Quand visible :** `availableAngles.length > 1` seulement (D-11)
- **Quand masquée :** un seul angle disponible OU `availableAngles` vide
- **Position :** sous `.imageWrapper` dans la colonne gauche (desktop 640px+) / sous l'image pleine largeur (mobile) — D-01
- **Orientation :** `flex-direction: row`, `gap: var(--spacing-xs)` (4px)
- **Margin-top :** `var(--spacing-sm)` (8px) séparation visuelle avec l'image principale — D-04 (fond intégré, pas de bandeau)
- **Scroll mobile :** `overflow-x: auto` si dépassement (D-03), scrollbar masquée visuellement (`scrollbar-width: none` Firefox, `::-webkit-scrollbar { display: none }` Chrome/Safari)

### Thumbnail individuel `.thumbnail`

- **Taille :** `width: 72px; height: 54px` (ratio 4/3 — wireframe Section 5) — D-02 (pas de label texte)
- **Position :** `position: relative` (requis pour `<Image fill />`)
- **Radius :** `var(--radius-sm)` (4px)
- **Border :** `3px solid transparent` au repos
- **Overflow :** `overflow: hidden` (clip l'image)
- **Cursor :** `pointer`
- **Background :** `none` (bouton natif reset)
- **Padding :** `0`
- **Hover :** `box-shadow: var(--shadow-md)` — `0 4px 6px rgba(0,0,0,0.07)`
- **Transition hover :** `box-shadow var(--transition-fast)` (300ms — acceptable pour hover, différent du crossfade image)
- **`flex-shrink: 0`** (évite la compression dans le scroll horizontal)

### Thumbnail actif `.thumbnailActive`

Exactement le même traitement que `.swatchSelected` Phase 8 (D-05) :
- `border-color: var(--color-primary)` — `#E49400` 3px
- `outline: 2px solid var(--color-primary)`
- `outline-offset: 2px`

### Image principale — Crossfade `.imageMain`

- **Animation :** `@keyframes imageFadeIn` — `from { opacity: 0 }` → `to { opacity: 1 }`
- **Durée :** `200ms ease` — NE PAS utiliser `--transition-fast` (300ms) — D-06 spécifie ~200ms
- **Déclenchement :** changement de `key={displayImageUrl}` sur `<Image>` — remount React = re-fade sans state supplémentaire
- **Classe :** `className={styles.imageMain}` appliquée sur le `<Image>` dans `.imageWrapper`

### Source des images thumbnail

- Les thumbnails affichent **toujours** `model_image.image_url` (photo originale du modèle)
- Jamais le rendu IA généré dans les thumbnails — seule l'image principale change (anti-pitfall 5 RESEARCH.md)

## Accessibilité — Contrat ARIA

| Élément | Attribut | Valeur |
|---------|----------|--------|
| Rangée thumbnails | `role="radiogroup"` | — |
| Rangée thumbnails | `aria-label` | `"Choisir l'angle de vue"` |
| Thumbnail individuel | `role="radio"` | — |
| Thumbnail individuel | `aria-checked` | `selectedAngle === img.id` → `"true"` / `"false"` |
| Thumbnail individuel | `aria-label` | `"Vue {img.view_type}"` — ex : `"Vue 3/4"`, `"Vue face"`, `"Vue profil"` |
| Image thumbnail | `alt` | `""` (décorative — le label est sur le `<button>`) |
| Image principale | `alt` | Voir règle D-07 ci-dessous |

**Alt text image principale (D-07) :**
- Avec tissu + angle : `"Canapé {model.name} en tissu {fabric.name} — vue {view_type}"`
- Sans tissu + avec angle : `"Canapé {model.name} — vue {view_type}"`
- Sans tissu + sans angle : `"Canapé {model.name}"`

**Focus clavier (discrétion Claude) :**
- Navigation Tab entre les thumbnails (comportement natif `<button>`)
- Activation Enter/Space sur le thumbnail focused (comportement natif `<button>`)
- Pas de roving tabindex — la rangée est courte (≤5 angles wireframe)

Source : CONTEXT.md D-07 + RESEARCH.md Pattern 6 + CLAUDE.md (messages en français)

## État initial et transitions d'état

| État | Image principale | Thumbnails affichés | Thumbnail actif |
|------|-----------------|--------------------|--------------------|
| Modal ouvert, pas de tissu | Photo originale angle 3/4 (ou premier) | Tous les `model_images` (photos originales) | Angle 3/4 (ou premier) |
| Tissu sélectionné, plusieurs angles publiés | Rendu IA angle actif | Angles ayant un rendu publié pour ce tissu uniquement | Angle conservé si disponible, sinon reset 3/4 |
| Tissu sélectionné, un seul angle publié | Rendu IA de l'unique angle | Rangée masquée (D-11) | n/a |
| Tissu désélectionné | Photo originale angle actif | Tous les `model_images` (retour D-13) | Angle conservé |
| Changement de modèle | Photo originale angle 3/4 du nouveau modèle | Tous les `model_images` du nouveau modèle | Angle 3/4 (reset D-16) |
| Modal rouvert même modèle | Image selon état précédent | Angles selon tissu précédent | Angle précédent préservé (D-15) |

## CSS à créer — Contrat classe par classe

Ajouter dans `src/components/public/Catalogue/ConfiguratorModal.module.css` :

```css
/* === Phase 9 — Navigation angles === */

.thumbnailRow {
  display: flex;
  flex-direction: row;
  gap: var(--spacing-xs);          /* 4px */
  margin-top: var(--spacing-sm);   /* 8px */
  overflow-x: auto;
  scrollbar-width: none;           /* Firefox */
}

.thumbnailRow::-webkit-scrollbar {
  display: none;                   /* Chrome/Safari */
}

.thumbnail {
  position: relative;
  flex-shrink: 0;
  width: 72px;
  height: 54px;
  border-radius: var(--radius-sm); /* 4px */
  overflow: hidden;
  border: 3px solid transparent;
  cursor: pointer;
  background: none;
  padding: 0;
  transition: box-shadow var(--transition-fast);
}

.thumbnail:hover {
  box-shadow: var(--shadow-md);
}

.thumbnailActive {
  border-color: var(--color-primary);
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}

/* Crossfade 200ms — NE PAS utiliser --transition-fast (300ms) */
.imageMain {
  animation: imageFadeIn 200ms ease;
}

@keyframes imageFadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}
```
