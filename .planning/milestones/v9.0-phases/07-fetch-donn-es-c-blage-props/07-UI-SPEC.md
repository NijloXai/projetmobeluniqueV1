### Règle d'erreur Phase 7

Si un des 3 fetches Supabase (`Promise.all`) échoue, le même message d'erreur existant s'affiche :

> "Impossible de charger les produits. Veuillez rafraichir la page."

Condition de déclenchement : `modelsResult.error || fabricsResult.error || visualsResult.error`
Approche : erreur totale (pas de dégradation partielle) — cohérent avec le pattern `CatalogueSection` actuel.
**Aucune nouvelle copie à écrire pour cette condition.**

### Aucune action destructive en Phase 7

Pas de suppression, pas de confirmation requise.

## Interaction Contract

### ConfiguratorModal — props inchangées visuellement

Le modal reçoit 2 nouvelles props (`fabrics`, `visuals`) mais **le rendu HTML/CSS est identique à Phase 6**. L'utilisateur ne voit aucun changement.

| Prop | Type | Obligatoire | Visible utilisateur |
|------|------|-------------|---------------------|
| `model` | `ModelWithImages \| null` | oui | oui (existant) |
| `onClose` | `() => void` | oui | oui (existant) |
| `fabrics` | `Fabric[]` | **oui (obligatoire — pas optionnel)** | non (Phase 8) |
| `visuals` | `VisualWithFabricAndImage[]` | **oui (obligatoire — pas optionnel)** | non (Phase 8/9) |

Justification du caractère obligatoire : prévenir les `undefined.filter is not a function` en Phase 8 si les props sont déclarées optionnelles (Pitfall 4 de RESEARCH.md).

- Restauration focus au CTA déclencheur à la fermeture (`triggerRef`)
- Backdrop click ferme le modal

### Scroll lock iOS

## Notes pour le Planificateur et l'Exécuteur

1. **Aucun nouveau fichier CSS à créer.** Phase 7 est une phase de tuyauterie — les modifications visuelles appartiennent aux Phases 8 et 9.

2. **Le placeholder "Configurateur a venir" reste en place.** Il sera remplacé en Phase 8 quand les swatches tissu seront construits.

3. **Les props `fabrics` et `visuals` dans `ConfiguratorModal` doivent être déclarées obligatoires** (sans `?`), même si leur contenu n'est pas rendu en Phase 7.

4. **Hooks React :** Tout nouveau `useState` ajouté dans `ConfiguratorModal` doit être déclaré **avant** la ligne `if (!model) return null` (ligne 67 actuelle). Le commentaire `// IMPORTANT : return null APRES tous les hooks` est déjà présent dans le code.

5. **Filtrage `is_active` :** Côté JS uniquement après le fetch — `.eq('fabric.is_active', true)` dans PostgREST n'a aucun effet sur les jointures (confirmé dans `/api/models/[slug]/visuals/route.ts`).
