## Composant Inventory

Nouveaux elements visuels a implementer dans `ConfiguratorModal.tsx` et `ConfiguratorModal.module.css`. Tous s'inserent dans `simulationState === 'done'` — stub actuellement vide (Source : 12-CONTEXT.md Code Context).

### 1. Zone resultat — etat `done` (colonne gauche)

Structure :
- **Container image resultat :** `position: relative`, `width: 100%`, `aspect-ratio: 4/3`, `border-radius: var(--radius-lg)` (12px), `overflow: hidden` — coherent avec `.previewContainer` existant
- **`<img src={resultBlobUrl}>`** : `width: 100%`, `height: 100%`, `object-fit: cover`, animation `imageFadeIn 400ms ease` (D-04 — fondu depuis progression)
- **Transition entree (D-04) :** La progression passe a 100% (deja implemente), puis `simulationState` passe a `'done'`, declenchant l'animation `imageFadeIn` sur le container resultat

### 2. Disclaimer IA (D-02)

- Position : directement sous l'image resultat (hors du container `aspect-ratio`)
- Style : `font-size: var(--font-size-sm)` (14px), `font-weight: 400`, `color: var(--color-muted)` (#888888), `font-style: italic`
- Margin-top : `var(--spacing-sm)` (8px)
- Copy exacte : `"Apercu genere par IA — le rendu reel peut varier"`

### 3. Bloc 4 boutons d'action (D-05)

Position : sous l'image resultat + disclaimer, dans la colonne gauche (D-08 mobile) ou colonne droite (desktop).

**Note layout :** Sur desktop (>= 640px), les 4 boutons s'affichent dans la colonne droite (`body`) sous le titre/sous-titre. Sur mobile (< 640px), ils s'affichent sous le disclaimer dans le flux colonne unique.

#### Bouton 1 — "Telecharger" (D-05, D-06)

- Height : 48px, width : 100%
- Font : `--font-size-sm` (14px), weight 600
- Fond : `var(--color-primary)` (#E49400)
- Couleur texte : `var(--color-text)` (#1D1D1B)
- Border : none, `border-radius: var(--radius-md)` (8px)
- Icone : `<Download size={16}>` (lucide-react), `aria-hidden="true"`, gap `var(--spacing-xs)` (4px)
- Comportement : `URL.createObjectURL` + `<a download="mobel-unique-simulation.jpg">` pattern (D-06)
- Focus ring : `outline: 2px solid var(--color-primary)`, `outline-offset: 2px`

#### Bouton 2 — "Partager" (D-05, D-07)

- Height : 48px, width : 100%
- Font : `--font-size-sm` (14px), weight 600
- Fond : `var(--color-whatsapp)` (#25D366) — couleur WhatsApp identifiable
- Couleur texte : `#FFFFFF`
- Border : none, `border-radius: var(--radius-md)` (8px)
- Hover : `var(--color-whatsapp-hover)` (#1FB855)
- Icone : `<Share2 size={16}>` (lucide-react), `aria-hidden="true"`, gap `var(--spacing-xs)`
- Comportement (D-07) :
  - Si `navigator.share` disponible ET `navigator.canShare({ files: [blob] })` : `navigator.share({ files: [file], title: 'Ma simulation Möbel Unique' })`
  - Sinon (desktop) : ouverture `https://wa.me/?text=...` avec message pre-rempli dans nouvel onglet
- Message WhatsApp pre-rempli : `"Regardez comment ce canape s'integre dans mon salon ! Visualise avec Mobel Unique — {model.shopify_url ?? 'https://mobelunique.fr'}"`
- Focus ring : `outline: 2px solid var(--color-primary)`, `outline-offset: 2px`

#### Bouton 3 — "Commander sur Shopify" (D-05)

- Height : 48px, width : 100%
- Font : `--font-size-sm` (14px), weight 600
- Fond : `transparent`
- Couleur texte : `var(--color-text)` (#1D1D1B)
- Border : `1px solid var(--color-text)` — style identique au `.ctaShopify` existant
- `border-radius: var(--radius-md)` (8px)
- Hover : `background: var(--color-text)`, `color: var(--color-background)` — identique au `.ctaShopify` hover
- Icone : `<ExternalLink size={16}>` (lucide-react), `aria-hidden="true"`, gap `var(--spacing-xs)`
- Rendu : `<a href={model.shopify_url} target="_blank" rel="noopener noreferrer">`
- Condition : **masque si `model.shopify_url === null`** (Claude's Discretion — coherent avec le pattern `ctaShopify` existant en Phase 8)
- Focus ring : `outline: 2px solid var(--color-primary)`, `outline-offset: 2px`

#### Bouton 4 — "Essayer une autre photo" (D-05, D-09)

- Height : 44px (hauteur reduite — action tertiaire), width : 100%
- Font : `--font-size-sm` (14px), weight 600
- Fond : `transparent`
- Couleur texte : `var(--color-muted)` (#888888)
- Border : none
- `border-radius: var(--radius-md)` (8px)
- Hover : `color: var(--color-text)` (#1D1D1B)
- Icone : `<RefreshCw size={14}>` (lucide-react), `aria-hidden="true"`, gap `var(--spacing-xs)`
- Comportement (D-09) : reset `simulationState → 'idle'`, efface `resultBlobUrl` + `selectedFile`, preserve config tissu/modele
- Focus ring : `outline: 2px solid var(--color-primary)`, `outline-offset: 2px`

**Gap entre boutons :** `var(--spacing-sm)` (8px) vertical — tous empiles en flex column.

### 4. Colonne droite — etat `done`

Sur desktop (>= 640px), la colonne droite (`.body`) affiche :
- Bouton retour `← Modifier la configuration` (herite de Phase 11 `.backButton` — inchange)
- Titre `h2` : "Votre simulation" (D-03), style `.simulationTitle` existant
- Sous-titre dynamique : `"{model.name} × {fabric.name} dans votre salon"` si tissu selectionne, sinon `"Canape original dans votre salon"` (D-03)
  - Font : `--font-size-base` (16px), weight 400, `--color-muted`, margin-top `var(--spacing-xs)` (4px)
- Les 4 boutons d'action (flex column, gap `var(--spacing-sm)`, margin-top `var(--spacing-xl)`)

Sur mobile (< 640px), la colonne droite affiche uniquement le titre + sous-titre. Les boutons d'action se placent dans la colonne gauche, sous le disclaimer.

## Layout responsive

Herite du layout 2 colonnes de `ConfiguratorModal` (Phase 6/9/11). Pas de changement de breakpoints.

| Breakpoint | Layout resultat | Colonne gauche | Colonne droite |
|------------|-----------------|----------------|----------------|
| `< 640px` (mobile) | 1 colonne empilee | Image resultat + disclaimer + 4 boutons d'action (D-08) | Titre + sous-titre en haut (avant image) |
| `>= 640px` (desktop) | 2 colonnes 50/50 | Image resultat + disclaimer | Retour + titre + sous-titre + 4 boutons d'action |

**Mobile :** Sur mobile, l'ordre de flux est : titre (colonne droite remontee) → image → disclaimer → boutons. Implementation via `order` CSS ou restructuration JSX conditionnelle.

**Approche retenue (Claude's Discretion) :** Sur mobile, les boutons s'affichent dans la colonne gauche apres le disclaimer. La colonne droite n'affiche que titre + sous-titre en mobile. Pas de `order` CSS — le JSX conditionne directement le rendu par breakpoint via `simulationState === 'done'`.
