## Interaction Contract

### Filtre temps réel

- Déclencheur : `onChange` sur l'input — chaque frappe
- Pas de debounce (D-06 — 20-30 produits en mémoire, zéro coût de render)
- `filteredModels` est une valeur dérivée calculée à chaque render — pas de `useState` supplémentaire ni de `useEffect`
- Normalisation des deux opérandes : `normalize(m.name).includes(normalize(query))`
- Fonction `normalize` : `str.normalize('NFD').replace(/[\u0300-\u036f]/g, '').toLowerCase().trim()`

### Bouton clear (X) dans le champ

- Apparaît : quand `query !== ''`
- Disparaît : quand `query === ''`
- Action : `setQuery('')` + `inputRef.current?.focus()` (retour focus sur l'input)
- Pas de transition d'apparition — swap instantané (cohérence avec le filtre instantané)

### Bouton "Effacer la recherche" (état vide)

- Visible uniquement dans l'état vide de recherche
- Action identique au bouton X : `setQuery('')` + focus sur l'input
- Style : bouton secondaire (outline) — visuellement moins dominant que le CTA principal de ProductCard

### Focus ring

- Champ de recherche : `box-shadow: 0 0 0 2px var(--color-primary)` au `:focus`
- Bouton X et bouton reset : focus visible via `outline: 2px solid var(--color-primary); outline-offset: 2px`

### Animation / transitions

- Filtrage des cards : **instantané** — pas de fade, pas de transition CSS sur les cards. Le filtre doit sembler immédiat et réactif.
- Apparition/disparition du bouton X : instantanée (pas de transition opacity)
- Raison : cohérence avec la décision D-06 (filtre instantané = sentiment de réactivité). Toute animation de fade ralentirait la perception.
