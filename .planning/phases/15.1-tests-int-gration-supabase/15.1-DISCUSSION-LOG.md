## Environnement de test

| Option | Description | Selected |
|--------|-------------|----------|
| Supabase CLI local (Docker) | supabase start lance une instance locale. Tests rapides, isolés, répétables. Docker requis. | ✓ |
| Projet Supabase de test | Projet distant dédié aux tests. Plus simple mais plus lent, dépend du réseau. | |
| Mocks améliorés | Enrichir les mocks actuels. Pas de vraie intégration mais zéro dépendance externe. | |

**User's choice:** Supabase CLI local (après clarification sur MCP vs CLI — MCP branching est pour usage ponctuel, CLI est le standard pour tests automatisés)
**Notes:** L'utilisateur a d'abord demandé si on pouvait utiliser le MCP Supabase. Explication : MCP tools sont disponibles dans Claude Code mais pas au runtime Vitest. CLI local est meilleur pour des tests automatisés (rapide, gratuit, offline, CI-friendly).
